# SDLC Model Justifications by Shubh Malhotra

Below is a list of each project scenario, the recommended SDLC model, and a justification explaining why that model is appropriate.

---

## #1: New e-Commerce Site  
**Recommended SDLC Model:** Agile (Scrum / Incremental)  
Agile is well-suited for this project because features that customers see like customization and outfit matching are likely to evolve based on user feedback. Incremental development allows the team to deliver core functionality early while coordinating extensions with a separate group.

---

## #2: Dental Office Site  
**Recommended SDLC Model:** Agile (Incremental)  
The website begins with simple, well-understood requirements but is expected to grow over time with additional features. An incremental Agile approach allows for gradual delivery and enhancement without strict deadlines.

---

## #3: Nuclear Power Plant  
**Recommended SDLC Model:** V-Model  
This software is safety-critical and must be thoroughly tested and validated at every stage of development. The V-Model ensures the following of requirements and verification activities, which is necessary for long-term reliability and safe behavior in case anything happens.
